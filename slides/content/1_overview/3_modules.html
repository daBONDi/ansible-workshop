Module

  • Module ist Code der auf das Zielsystem kopiert wird und dort im ausgeführt wird.
  • Mehrfache Ausführung sollte immer zum selben Ergebnis führen(idempotent)
    • Es gibt Module die NICHT idempotent sind! Modul Doku beachten!

Linux

  • Fast alle Module sind in Python geschrieben
  • Python + benötigen Python Package müssen am Target Host vorhanden sein!

Windows

  • Powershell Scripts
  • Werden per WinRM kopiert und ausgeführt
  • Benötigte Powershell Module müssen vorhanden sein

# Module: Commands Wenn es kein Ansible Modul gibt, können normale Shellbefehle/Scripts ausgeführt werden. - **command / win_command:** Einfaches Command(Keine Shell) - **shell:** Führt das Command durch eine Shell auf zb. /bin/sh und kann dadurch auch Pipen(**Hier aufpassen**) - **script/win_script:** Führt das lokale Script auf den Remote Host aus - **raw:** Führt das Command aus OHNE durch das Ansible Modul Subsystem durchzulaufen > - **Sind nicht idempotent!** > - Sollte nur als letzte Möglichkeit verwendet werden! > - Ansible Module schreiben ist nicht schwer! :-)

Meist verwendete Module

Linux

  • copy Kopiert von Controller zu Host
  • file Stellt sicher das eine Datei/Ordner/Berechtigung vorhanden ist
  • template Baut zb. eine Config File mit Jinja2 Templating zusammen
  • apt/yum Package Manager
  • service Linux Daemon Management
  • ...

Windows

  • win_copy Windows Module für copy
  • win_file Windows Module für file
  • win_template Windows Module für Template
  • win_chocolatey Chocolatey Package Manager
  • win_acl /win_acl_inheritance Disk/Registry ACL
  • win_serviceWindows Service Management
  • ...
