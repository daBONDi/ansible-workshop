Inventory

  • Beschreibt die Hosts(Nodes)
    • IP/DNS/Connection Setup(SSH/WinRM)
    • Host spezifische Variablen
  • Gruppen
    • Ein Host kann in mehreren Gruppen sein(zb. frontend, application, database)
    • Gruppen können auch verschachtelt werden

Quelle

  • Statisch File (INI, YAML, JSON)
  • Dynamisch
    • Inventory Plugin(zb. Netbox, AWS, Azure)
    • Shell Tool das INI/YAML/JSON in stdout ausgibt
  • Angabe als String "192.168.1.1,192.168.1,"
- das komma ist kein Tippfehler, es muss eine Liste ein - gegen die Hosts und Gruppen werden dann die Plays/Tasks ausgeführt
# Statisches Inventory ```ini 10.0.1.21 10.0.1.22 10.0.1.41 host.example.com ```
# Statisches Inventory - Beispiel ```ini [control] # inventory_hostname | ansible_host ctrl ansible_host=10.0.1.10 my_custom_var=hello [web] debian[1:5] ansible_host=10.0.1.[21:25] [linux:children] web control [linux:vars] ansible_user=admin ansible_ssh_private_key_file=~/.ssh/workshop.key ``` ### inventory_hostname Der für Ansible interner festgelegter Name(für Filterung, Playbook Host Angaben usw.) ### ansible_host Der für die Verbindung verwendeter Name/IP Adresse notes: - Unterscheid zwischen - Angabe von [1:5] - geschachtelte Gruppen - Gruppen Variablen - Host variablen my_custom_var=hello
# Inventory Testen/Abfragen - ansible-inventory ```bash # Zusammengebautes Inventory abfragen und in YAML format ausgeben ansible-inventory --list -i inventory.ini -y ``` ```bash # Nur für einen host ansible-inventory --host=debian1 -i inventory.ini -y # Output ansible_host: 10.0.1.10 ansible_port: 22 ansible_ssh_common_args: -o StrictHostKeyChecking=no ansible_ssh_private_key_file: ~/.ssh/workshop.key ansible_user: admin ``` notes: - verarbeitete Inventory ansehen und kontrollieren - überprüfen wenn gruppen sehr verschachtelnt sind - settings von dynamic inventory plugins überprüfen
