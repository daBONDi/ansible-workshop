Ansible Tower / AWX

Ansible Tower (Enterprise Stable) https://www.ansible.com/products/tower
AWX (Open Source Unstable) https://github.com/ansible/awx

  • Ansible Tower kann man verwenden
  • Enteprise Features (SAML, LDAP Auth, Role based access)
  • AWX für Produktion derzeit noch nicht Brauchbar
  • AWX = Docker Container, Migration Sucks

ARA

https://ara.recordsansible.org/

  • Schönes Logging inkl. Module Input sowie Output und Result
  • Logging von mehreren Ansible Controllern in eine ARA Datenbank möglich
  • Tolles Logging
  • Kann von mehreren Ansible Controllern Verwendet werden
  • Ansible Callback Plugin Sended die Results und Exection zum ARA Server

Rundeck - Job Engine (Java)

https://www.rundeck.com/open-source

Enterprise Runbook Execution Engine mit Unterstützung für Ansible

  • Java Based Service
  • Enterprise Features in der Open Source Variante
  • Tolle Runbook Eingabe Form
  • Advanced cronjob Tool

Stackstorm

Eventdriven Enterprise Runbook Execution Engine

https://stackstorm.com/

  • Python Service
  • Event Driven, mit Message Bus
  • Ein API Call kann mehrere Runbooks Starten usw.
  • Opensource Variante hat kein Rolebased Access
  • Opensource Variante nur simples login
  • HA möglich
  • Playbooks sind auch in YAML geschrieben
  • Passt besser zum Ansible Ökosystem finde ich, da Python und Playbooks sind auch in Yaml geschrieben

Node-CI / Ansible UI

Simple NodeJS Anwendung/CI für das Ausführen von Ansible Playbooks

  • Sehr simples REST Interface für das ausführen von Playbooks
  • Erweiterbar mit anderen Node-CI Plugins
  • Für simple sachen mehr als ausreichend

Molecule - Testing Ansible Roles

Test Engine für Ansible Rollen mit Docker/Vagrant Support

https://molecule.readthedocs.io/en/stable/

  • Super zum einbinden in eine CI
  • Komplette Tests / Smoketest / Integration Test möglich
  • Ansible Community Role Testing Framework
  • Nur wirklich auf Linux nutzbar / in WSL sehr quirky
  • Arbeitet sehr gut mit Vagrant und Docker zusammen
