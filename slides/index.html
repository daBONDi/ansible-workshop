#### Was ist Ansible Eine Automatisierungs Engine
#### Ansible ist Einfach - Lesbar (YAML) - Keine speziellen Programmierkenntnisse (Module) - Aufgaben werden in Reihenfolge ausgeführt > BUZZWORD: **Get productive quickly**
#### Ansible ist Mächtig - Provisioning - Application Deployment - Configuration management - Orchestration - Continous Delivery - Security & Compliance - ... > BUZZWORD: **Orchestrate the app lifecycle**
#### Ansible ist Agentless! - Linux/Network = OpenSSH - Windows = WinRM - Keine Sicherheitslücken im Agent - Kein Agent zu Patchen - ***Kein Agent auf Kundensystemen*** > BUZZWORD: **Predictable, reliable and secure**

#### Warum Ansible - Einfach zu Lesen und zu arbeiten - Automatisierung läuft immer von oben nach unten - Modular Entwickelt und einfach erweiterbar - Kein Server benötigt - Kann am Techniker Client laufen notes: von oben nach unten im gegensatz zu salt/chef (Kombinierte form)
